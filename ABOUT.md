# About This Project

## Project Title
**Cuddle Cup POS and Inventory System**

## What This System Is
This is a **desktop-based shop management system** made for a small store or cafe setup.

It helps the business handle:
- user login
- cashier sales transactions
- receipt viewing
- product and inventory management
- transaction monitoring
- admin and cashier account handling

This project was built as a **school project system** with a real working setup, not just a design or mockup.

## Who Can Use It
There are only **2 user roles** in the system:

### Admin
The Admin can:
- view the dashboard
- manage products
- restock inventory
- manage cashier accounts
- view transaction history
- review receipts and store activity

### Cashier
The Cashier can:
- log in to the POS
- choose products
- add items to the cart
- accept payment
- compute change
- complete a sale
- show the receipt

## Technology Used
The system uses simple and practical technologies:

### 1. Java
Java is the main programming language used to build the whole system.

Why it was used:
- stable and widely taught in school
- good for desktop applications
- organized and reliable for business logic

### 2. Java Swing
Java Swing is the tool used for the **user interface** or the actual windows, buttons, forms, tables, and dashboard screens.

In simple terms:
- this is what makes the system look like a real desktop application
- it is used for the login page, admin screens, cashier page, and receipt view

### 3. MySQL
MySQL is the **database** of the system.

It stores important information such as:
- user accounts
- products
- stock quantities
- transactions
- receipts
- activity logs

### 4. JDBC
JDBC stands for **Java Database Connectivity**.

This is the bridge between:
- the Java program
- and the MySQL database

It allows the system to save and read data properly.

### 5. XAMPP
XAMPP is used because it includes **MySQL** and makes local setup easier during development.

This means the project can be run on a school or personal computer without needing a paid online server.

## What Kind of System It Is
This is a:
- **local desktop application**
- **database-connected system**
- **role-based system**

It is **not** a website and **not** a mobile app.

- clean

If explained in one sentence:

> The system is a Java Swing desktop POS application connected to a MySQL database for managing sales, products, inventory, receipts, and user accounts.
